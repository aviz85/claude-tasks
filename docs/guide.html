Get Started

Two ways to build your own GTD system with Claude Code

Recommended

DIY: Prompt Claude Code

Start fresh and let Claude Code create a custom task management system tailored to your workflow. Just paste the prompt below.

~5 minutes
1

Open Claude Code

In your terminal or preferred environment

2

Paste the prompt

Copy the prompt below and send it to Claude

3

Start using!

Claude will create the system and explain how to use it

Prompt 
Create a GTD (Getting Things Done) task management system for me.

Requirements:
- Use markdown files for storage (tasks/inbox.md, tasks/today.md, tasks/backlog.md, tasks/someday.md)
- Create Claude Code commands: /add, /today, /done, /standup, /review, /focus, /inbox, /prioritize
- Create skills for: task-parsing (natural language to structured tasks), priority-matrix (Eisenhower matrix), time-estimation
- Task format: "- [ ] Task @project #tag !priority ~estimate"
- Priorities: !high, !medium, !low
- Time estimates: ~15m, ~30m, ~1h, ~2h, ~4h

Create the file structure, CLAUDE.md documentation, and all necessary skill/command files. Start with a brief explanation, then create everything.
Quick Start

Clone the Repo

Get the complete system instantly. All commands, skills, and file structure ready to go.

~1 minute
1

Clone the repository

Run the command below in your terminal

2

Rename example files

Copy .example.md files to .md

3

Run /standup

Start your first daily planning session

Terminal 
# Clone the repo
git clone https://github.com/aviz85/claude-tasks.git
cd claude-tasks

# Set up task files
cp tasks/inbox.example.md tasks/inbox.md
cp tasks/today.example.md tasks/today.md
cp tasks/backlog.example.md tasks/backlog.md
cp tasks/someday.example.md tasks/someday.md

# Start using!
claude /standup
claude-tasks/
├── .claude/
│ ├── commands/ — /add, /done, /standup...
│ ├── skills/ — parsing, prioritization
│ └── agents/ — daily-standup, reviewer
├── tasks/
│ ├── inbox.example.md
│ ├── today.example.md
│ └── ...
└── CLAUDE.md — instructions
